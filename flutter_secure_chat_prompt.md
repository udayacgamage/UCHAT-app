## 🔐 Security Implementation

### 1. End-to-End Message Encryption (AES-256)

```dart
// lib/services/encryption_service.dart
// Generate a unique AES key per chat (derived from both user IDs)
// Store the key in flutter_secure_storage
// Encrypt every message body BEFORE writing to Firestore
// Decrypt every message body AFTER reading from Firestore
// Never store the raw encryption key in Firestore
```

Implement these methods:
- `String encryptMessage(String plainText, String chatId)`
- `String decryptMessage(String cipherText, String chatId)`
- `Future<void> generateAndStoreChatKey(String chatId)`
- `Future<String?> getChatKey(String chatId)`

### 2. Biometric Authentication
- Use `local_auth` package
- On app resume from background (after 3+ minutes), require biometric or PIN
- Show a lock screen with biometric prompt button

### 3. Firestore Security Rules
Generate complete security rules enforcing:
- Users can only read/write their own user document
- A user can only read a chat document if they are a participant
- A user can only send messages to chats they are participants of
- Message deletion only by the sender or chat admin
- Group creation requires auth

### 4. Secure Storage
- All sensitive tokens (FCM token, auth tokens) stored via `flutter_secure_storage`
- No sensitive data stored in SharedPreferences

## ⚙️ Key Features to Implement

### Authentication Flow
1. Check if user is logged in on splash
2. On login: fetch/create user document in Firestore, update FCM token, mark isOnline = true
3. On logout: mark isOnline = false, clear secure storage tokens
4. Handle auth state changes via `StreamProvider` using `authStateChanges()`

### Real-time Messaging
- Use `snapshots()` stream on messages collection — ordered by `createdAt` descending
- Paginate: load 30 messages initially, load more on scroll to top
- Show delivery receipts: update `deliveredTo` array when message received
- Show read receipts: update `readBy` array when chat screen is open
- Update unread count in the chat document when new message arrives and user is NOT in that chat screen

### Typing Indicator
- Write to `/chats/{chatId}/typing/{uid}` with a timestamp when user is typing
- Delete that document when user stops typing (debounce 1.5s)
- Listen to the typing collection and show animated 3-dot indicator

### Online/Offline Status
- Set `isOnline = true` and update `lastSeen` when app is in foreground
- Set `isOnline = false` and update `lastSeen` when app goes to background
- Use `AppLifecycleState` listener in `app.dart`

### Push Notifications
- Request permission on first login
- Send notification via FCM when a message is sent to a user who is offline
- Tap on notification → navigate directly to that chat screen
- Handle foreground notifications with an in-app banner

### Image & File Sharing
- Pick image from gallery or camera
- Compress image before upload (max 800px width, 70% quality)
- Upload to Firebase Storage at `chats/{chatId}/{uuid}.jpg`
- Show upload progress in the message input bar
- Display images with `CachedNetworkImage` with a blur placeholder
- Tap image → full-screen viewer with pinch-to-zoom

### Message Actions (long press)
- **Reply**: show reply preview in input bar, store `replyTo` in message
- **Copy**: copy decrypted text to clipboard
- **Delete for me**: soft delete (hide from UI, keep in Firestore)
- **Delete for everyone**: set `isDeleted = true`, show "This message was deleted" placeholder
- **Forward**: open contact picker, send to selected chat

### Search
- Search users by name or email (Firestore `where` query with `>=` / `<=` for prefix search)
- Search messages within a chat (local search on cached messages)
